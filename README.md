數獨專案進度報告：程式碼功能詳解


專案概述

本專案採用 MVC (Model-View-Controller) 的簡化架構，將「資料運算」與「介面控制」分離。程式共分為五個檔案，具備固定題目、即時正誤判斷、以及三次錯誤限制等核心功能。


檔案功能詳細說明

A. Sudoku.h (邏輯核心標頭檔)

角色：定義數獨的「資料結構」與「運算介面」。

關鍵功能：

定義 SIZE = 9：固定數獨維度。

聲明 grid (玩家盤面) 與 solution (正確答案) 兩個二維陣列。

定義 validateAndSet 函式原型：這是本程式最核心的邏輯接口。


B. Sudoku.cpp (邏輯核心實作檔)

角色：負責數獨資料的儲存與驗證邏輯。

核心功能：

資料初始化：在建構子中直接以硬編碼 (Hard-coded) 方式填入一組完整的數獨解答 (sol) 與對應的題目 (puzzle)。

即時驗證機制 (validateAndSet)：

當玩家嘗試填入數字時，程式會立即比對該座標在 solution 陣列中的數值。

只有當 輸入值 == 正確答案 時，才會更新 grid 陣列並回傳 true；否則不動作並回傳 false。

完成狀態檢查 (isComplete)：遍歷盤面，確認是否所有格子都已填滿。


C. Game.h (遊戲控制器標頭檔)

角色：定義遊戲的「狀態」與「流程控制方法」。

關鍵屬性：

board：包含 Sudoku 類別的物件。

wrongCount：核心功能變數，用於追蹤玩家目前答錯的次數。


D. Game.cpp (遊戲控制器實作檔)

角色：負責處理所有的「使用者互動」與「畫面顯示」。

主要模組：

介面繪製 (display)：動態生成棋盤線條與座標。

UI 優化：上方與左側均標註 1-9 的編號，並移除右側多餘提示，方便玩家對齊座標。

遊戲主迴圈 (start)：

座標轉換：將玩家輸入的 1-9 座標轉換為程式內部的 0-8 索引。

錯誤懲罰機制：若 validateAndSet 回傳失敗，wrongCount 會加 1。

終止條件：當玩家完成數獨，或 答錯次數達到 3 次 時，迴圈自動終止。

輸入除錯：利用 std::cin.clear() 與 ignore() 處理非數字的非法輸入，避免程式崩潰。


E. main.cpp (程式進入點)

角色：啟動器。

功能：實例化 Game 物件並呼叫 start() 方法，啟動整個遊戲生命週期。
